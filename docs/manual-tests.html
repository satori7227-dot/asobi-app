T1. SuggestionFormView — Webhook 失敗時の mailto フォールバック

Discord Webhook が落ちている／ネットワーク無し時に、標準メーラー経由で satori7227@gmail.com 宛に意見を届けられることを確認する。

事前準備

  • Constants.feedbackWebhookEnabled = true（既定値）
  • テスト時に Webhook URL を意図的に無効値に差し替えるか、機内モードで送信を試す
  • Simulator なら「Settings → Mail」でテスト用 Apple ID をメール設定済にしておくとリンク開ける

手順

  1. アプリ起動 → ScenePicker 右上の💡 (アイデア箱) をタップ
  2. 「種類」を選び、シーンを選び、「例: 〜」のフォーマットで本文を入力
  3. 機内モード ON（または Webhook を意図的に破壊した debug ビルド）
  4. 「送信する」ボタンをタップ
  5. 失敗状態のオレンジバナーが出ることを確認
  6. 「メールで送る」リンクをタップ
  7. 標準メーラーが起動、To: satori7227@gmail.com、件名 [ASOBI] バグ / 飲み会 等、本文に入力したテキストが入っていることを確認

T2. ReviewPrompt — 条件達成時の SKStoreReviewController 発火

「初回起動から 7日以上 + 起動 3回以上 + 5本消化 + 同バージョン未表示 + 前回から120日以上」のAND条件で レビュー要請ダイアログが出るかを確認する。

テスト用に条件を満たす方法

  • UserDefaults を Settings.app またはデバッグコンソールから直接編集:
    • asobi.firstlaunch.date.v1 を 8日前に
    • asobi.launch.count.v1 を 3 に
    • asobi.totalplayed.v1 を 5 に
  • または、King's Game 配布完了 → 詳細を閉じる動線を 5 回繰り返す（recordCompletedGame が累積）

確認ポイント

  1. GameDetail の「閉じる」または King's Game 配布完了でレビュー要請ダイアログが出る
  2. 同セッション内では一度しか出ない
  3. 一度表示されたら、同バージョンでは再度出ない

T3. 多言語化 — UI 英語切替の動作

Settings.app で iPhone の言語を English にした時、ScenePicker・ContextInput・Proposal の UI と海外80件のゲーム名が英訳されることを確認する。

  1. Settings.app → General → Language & Region → Language を English にする
  2. ASOBI を再起動

  4. テンション/所要時間が「Calm」「5–15 min」等に切替わる
  5. 海外由来ゲームの詳細を開いて、name/summary/rules が英語表記になっていることを確認
