Color Asset コントラスト監査

対象: Asobi/Resources/Assets.xcassets/SceneColors/ 配下の 13 シーン色 + Launch/Accent 色。
目的: ダークモードで accent 色とテキストのコントラスト比が WCAG AA (4.5:1) を満たすか確認、 シーンアイコンが読みにくくなる箇所を特定する。

監査ガイドライン

  • テキスト × 背景: 通常テキストは 4.5:1 以上、巨大文字（18pt 太字 / 24pt 以上）は 3:1 以上。
  • UI 要素 × 背景: ボタン枠線・アイコンなど非テキストは 3:1 以上。
  • Accent 色 × Card 背景: ScenePicker のシーンカードは accent 色塗りなので、白テキスト/黒テキストの両方で AA 達成を確認。

確認手順

  1. Xcode で Asobi/Resources/Assets.xcassets/SceneColors/<Name>.colorset を開く
  2. Inspector で Light / Dark の Hex 値を読み取り、上表に転記
  3. WebAIM Contrast Checker で
    • 背景 = 確認対象色 / テキスト = 白（#FFFFFF） / レベル AA Large 3:1 以上か？
    • 背景 = 確認対象色 / テキスト = 黒（#000000） / レベル AA Large 3:1 以上か？
  4. 結果欄に PASS / WARN / FAIL を記入
  5. FAIL があった色は SceneColors/<Name>.colorset の Dark 値だけ調整（Light はブランド優先で原則変えない）

Simulator での目視確認

  • Simulator → Features → Toggle Appearance（または Settings → Developer → Dark Appearance）でダークモード切替
  • ScenePicker のシーンカードを順番にスクロールし、accent と「シーン名」テキストの読みやすさを確認
  • Accessibility Inspector → Color & Contrast でも数値化可能（macOS のみ）
