2026-06-18 ・ レビュアー: Claude（Opus 4.7） ・ 対象: MVP v1 全体
総評： アーキテクチャと拡張性は良好（SwiftUI標準・@Observable・JSONベース・xcodegen自走）。ただし App Store 公開ブロッカーが複数、メモリリーク2件、アクセシビリティ未対応、 ゲームDB品質バラツキ がある。MVP として「身内 TestFlight 配布」までは十分動くが、 公開審査・万人公開には 3 段階の磨き込みが必要。総改善ポイント 43 件 を以下に整理。

Criticalgames.json 読み込み失敗時のサイレント失敗

🟠 High（早めに対応）

HighDynamic Type 未対応
至る所で .font(.system(size: 48, weight: .heavy)) 等の固定サイズ。 視覚障害者・高齢者向けの文字サイズ拡大が効かない。 .font(.largeTitle.weight(.heavy)) 等のセマンティックスタイルに置換すべき。
ScenePickerView/ContextInputView/GameDetailView 多数
HighVoiceOver ラベル不足
シーンカード・ツールカード・お気に入りボタン等にラベルなし。 画像アイコンのみのボタンは VoiceOver で「ボタン」としか読まれない。 .accessibilityLabel(...) 追加が必要（全画面）。
Highダークモード対応の不完全さ
Scene の色（accentHex: "#4F7CAC" 等）が固定 Hex。 ライトモード前提の彩度設定で、ダークモードだと飽和して読みにくい。 カラーアセット化 or .environment(\\.colorScheme) 分岐推奨。
Asobi/Models/Scene.swift:10-16
HighContextInput 戻り時の状態破棄
提案画面から戻ると ContextInputView の入力（人数・テンション）が初期値に戻る。 「シャッフルしたら全然違うので入力やり直したい」が頻発するが、毎回 4人/medium/short からリセット。 @SceneStorage or 親で State 保持して child に Binding 渡すべき。
Asobi/Views/ContextInputView.swift:6
Hightrailing-closure 内の弱参照欠如
Timer.scheduledTimer { _ in winner = ... } 等で self をキャプチャしているが、@Observable クラスなのでクロージャ持つ間は解放されない。 Sheet を頻繁に開閉するとインスタンスが残る可能性。[weak self] パターン or Timer 廃止が望ましい。
RoulettePickerView/TimerView
HighゲームDB の派生名重複
「ジェスチャーゲーム家族版」「NGワード旅行版」「イントロクイズ家族版」など、 名前変更しただけで内容ほぼ同じ派生がある。意見箱で 👎 が溜まる前に内部整理推奨。 20の質問 / 5秒ルール / 神経衰弱 / お絵描きしりとり は family/couple/travel 等に重複登録あり。
Asobi/Data/games.json
Highスプラッシュ画面・初期チュートリアル不在
初回起動時に「ツール」「お気に入り」「意見箱」の場所と意味が分からない。 ツールアイコンが die.face.5.fill だけだと「サイコロ？」と誤解される（実は全 6 ツールのハブ）。 オンボーディング 3 画面 or 初回 hint オーバーレイ推奨。
Highscenes 再分類の機械的すぎる割当

🟢 Medium（保守性・将来拡張）

Mediumマジックナンバー直書き
プレイヤー人数の上下限（2〜10）、サイコロ個数（1〜6）、ルーレット最低人数（2）など ハードコード。Constants.swift に集約すると保守性向上。
MediumDispatchQueue.main.asyncAfter の利用
ProposalView.refresh / CoinFlipView.flip で asyncAfter(deadline: .now() + 0.6) 等。 Swift Concurrency への置換推奨（Task { try? await Task.sleep(...) }）。
ProposalView.swift:72, CoinFlipView.swift:64
MediumLocalizable.strings 未作成
全 UI 文字列が日本語ハードコード。 英語版 App Store 申請する時点で localization 全書き直し。 今のうちに String(localized:) パターンに移行が安価。
MediumUserDefaults キーが各クラスに散在
"asobi.feedback.v1" 等の文字列が直書き。 enum or DefaultsKey 構造体に集約推奨。
FeedbackStore.swift:27
Mediumシーン7枚は縦リスト疲れ
トップ画面で 7 シーンが縦リスト。スクロール必須で「リモート飲み」は折り畳まれる。 2×4 グリッドにすれば一画面に収まる。
ScenePickerView.swift
Mediumtension の値域固定（calm/medium/high）
将来「お酒入った時」「子供向け穏やか」など新軸を入れたくなった時、 string raw value enum だと拡張しづらい。OptionSet 化 or 構造体化検討。
MediumProposalView「もう一度シャッフル」のみ
条件変更したい時、戻るボタンで戻って再入力。 「条件を変える」「シーンに戻る」ショートカット欲しい。
MediumFavoritesView でシーン情報を失う
game.scenes.first { ... } ?? GameScene.initial[0] でその場しのぎ。お気に入り保存時のシーン context を別途記録すべき。
FavoritesView.swift:50
Medium空状態の説明不足
ContentUnavailableView の本文が短く、何をすれば次に進めるかが弱い。 「条件を緩めるとヒットしやすい」など示唆が欲しい。
Mediumitems が紙コップ・風船等の物理道具と「お題リスト」を同じリストに入れている
表示・フィルタロジックで「アプリ提供」と「物理」を分離できているが、 データ的には混在。items: { physical: [], appProvided: [] } のような分離が綺麗。
Mediumduration の 3 段階が粗い
short/medium/long の 3 段階。「3分以内」「10分以内」のような実時間軸の方が ユーザーには直感的。整数の秒数フィールドにする選択肢あり。
MediumKing ゲームくじの「タップ&長押し」の意図不明
「ボタンをタップ」と「長押し」両方で表示するが、片方だけで十分。 HIG 的にはタップで開示が標準。長押しは別アクション（コピー等）に予約すべき。
KingGameDistributorView.swift:147
Mediumprompts.json のスキーマが手動メンテ前提
カテゴリ追加時に Models/Prompt.swift と PromptRepository.swift も同時更新が必要。 「PromptCollection」を辞書ベース（[String: [String]]）にすれば追加が JSON だけで完結。
Mediumエラーハンドリングの一貫性なし
try? JSONDecoder().decode を多用。失敗時に黙って nil や空配列。 デバッグ難。Logger 経由でエラーを記録すべき。
Mediumナビバーアイコンの意味不明
左⭐（お気に入り）、右🎲💡（ツール／意見箱）。 サイコロアイコン＝「ツール全般」を直感できない。 「wand.and.stars」（魔法の杖）や「hammer.fill」（道具箱）等の代替案検討。
Mediumアプリ名「あそぶ？」の登録可能性未確認
App Store 申請時に同名アプリ存在チェック必要。? 記号は審査で たまに難癖つく。「あそぼ」「アソビ」など案も持っておく。

⚪ Low（Nice to have）

Lowアニメーション fade のチラつき
ProposalView の refresh で visible=false → 0.05s 後 visible=true。 時々 1 フレームで再描画されて瞬きが見える。
Lowシーンカード長押し未活用
長押しで何も起きない。「シーンをお気に入りに固定」「シーン別の統計を見る」等の余地あり。
Low背景色の単調さ
トップが白一色。シーン選択時にうっすらシーンカラーで彩る等の演出余地あり。
Lowタイマープリセット少なめ
5/10/30/60/180/300 秒のみ。カスタム秒数入力 UI が無い。
Lowトランプの背面表示なし
山札を視覚的に表示しない（残り枚数テキストのみ）。 山札カードの裏面アイコンがあると体感的に「カード引いてる」感が出る。
Lowお気に入り並べ替えなし
FavoritesView は repo.games.filter の順（JSON 並び順）。 お気に入り追加日時順 or 名前順の切替欲しい。
Lowフィードバック理由収集なし
👎 だけ記録するが、却下理由を任意で書ける欄が無い。 PDCA に活かしにくい。
LowHaptics 未活用

📐 推奨ロードマップ

  • Phase 1（今週中・公開前必須） アプリアイコン作成、Launch Screen 整備、プライバシーポリシー作成、Timer リーク修正、JSON 読み込みエラー UI、mailto: フォールバック
  • Phase 2（来週・UX強化） Dynamic Type 対応、VoiceOver ラベル追加、初回オンボーディング、戻り時の入力保持、お題重複防止
  • Phase 3（来月・品質磨き） ゲームDB 整理（派生統合・scenes 精緻化）、Unit Test 整備、ダークモード調整、Haptics 追加、タイマー音
  • Phase 4（公開後） ローカライズ準備、お気に入りソート、Phase 2 ヘルメスエージェント、Supabase 移行

👀 良かったところ

  • ◎ SwiftUI 標準 API のみで構築、外部依存 0（メンテ安心）
  • ◎ @Observable + Environment で状態管理が綺麗（Repository/Store/Prompt の3層）
  • ◎ games.json と prompts.json だけでコンテンツ拡張可能（PDCA 高速）
  • ◎ xcodegen ベースで .xcodeproj をコミット不要（チーム開発時もコンフリクト 0）
  • ◎ matchedGeometryEffect / symbolEffect / phaseAnimator を効果的に活用
  • ◎ ツールセット 7 種が独立した View で疎結合（追加・修正が局所的）
  • ◎ ゲーム × ツール × お題の三層構造で「アプリだけで遊べる」体験を実現
本レビューは MVP v1 時点のスナップショット。今後の意見箱フィードバックや実機テストで 優先順位は動く。Phase 1 だけ Yes/No を決めれば、Phase 2 以降は逐次実装で十分。
