⑦-B 初回リリースで IAP を完全無効化する仕組み

StoreKit 未実装で IAP UI だけ動かすと App Store 審査で必ずリジェクト。Feature Flag で初回は IAP を完全に隠す仕組みを実装済み。

Asobi/Services/Constants.swift（既に作成済み）
enum Constants { /// プレミアム機能（IAP）の有効化フラグ /// - false: 初回リリース時。全シーン・全お題が無料で開放、ロックUI 非表示 /// - true: Phase β 以降。プレミアムシーン・お題に課金ゲート、StoreKit 連動 static let premiumEnabled = false }

初回リリース時の動作（premiumEnabled = false）：

  • 全 13 シーンが無料で開放（婚活・就活・同窓会含む）
  • 全 9 お題カテゴリが無料で開放（大人の本音・カップル本音含む）
  • 「オールインパック ¥1,200」のCTA非表示
  • PurchaseSheetView のロックUI は完全に表示されない

Phase β で課金導入する時：

  1. App Store Connect で IAP プロダクト登録（com.idogawa.Asobi.scene.konkatsu 等）
  2. StoreKit 2 で Product.products(for:) 実装
  3. Constants.premiumEnabled = true に変更
  4. App Store 更新申請
審査時の重要メッセージ：初回リリースは完全無料アプリとして申請。「IAP の説明や購入ボタンが見えない」状態で審査を通す。これでリジェクトリスクを 1 つ消せる。

⑦-C 罰ゲーム要素のリジェクト回避

App Store 説明文に以下の文言を入れて、「全員参加型・健全な範囲」を明示：

説明文の補足（既に追加済み）
全員参加型・健全な範囲で、シーンに合わせた選び方を心がけています。

追加的な配慮：

  • 「飲み会」→「会食 / 集まる場面」（アルコール強調を避ける）
  • 「罰ゲーム決め」シーン名は維持（一般的すぎて変更すると混乱）、ただし説明文では「順番・指名決め」と表記
  • 「王様くじ」ツール名は維持、説明文では「順番決め」と表記
  • 過激なゲーム（物理的暴力・拘束等）は含まない（既に除外済み）
  • アルコール強要を罰ゲームの中核にしない（既に未掲載）
