# ASOBI v1.1 ロードマップ v1（2026-06-24 起案）

v1.0 リリース（2026-09-15 iOS 27 GA 同期想定）後の追加機能と外部リサーチによる根拠。

## 機能スコープ

v1.1 は **iOS 27 / Apple Intelligence + Featuring 強化** をテーマとする。実装期間目安 4-6 週間（v1.0 リリース後 30-60 日）。

### 1. App Intents 3 件（最優先）

**理由**: WWDC26 で SiriKit が deprecated（2-3 年窓）。Heads Up! / どこパが現状 Siri 統合確認できず、ASOBI が先行で voice-first 差別化 + Innovation 軸の Featuring 点数獲得。

| Intent | trigger | action |
|---|---|---|
| `StartGameIntent` | 「Hey Siri, ASOBI でパーティーゲーム始めて」 | ScenePicker 起動・直近使用シーンで提案 |
| `NextPromptIntent` | 「次のお題」「もう一つ」 | PromptGachaView 内で次のお題に進む |
| `RandomGameIntent` | 「ASOBI でゲーム」 | 全シーン横断のランダム 1 件提案 |

実装：`AppIntents` framework + `IntentDescription` + `ParameterSummary` + Shortcuts 自動登録。

### 2. Live Activity「現在のお題」

**理由**: iOS 18 で Interactive Widget は標準 widget 比 **3× エンゲージメント**。Live Activities の party game 公式採用事例が空白 = **世界初級事例**を作れるニッチ。

仕様：
- ゲーム進行中に Live Activity 起動 → Lock Screen + Dynamic Island に現在のお題と「次へ」「スキップ」ボタン
- スマホを机置きしても画面 ON で次のお題が見える宴会場 UX
- 6 分の `staleDate` で自動終了

### 3. Foundation Models による語彙バリエーション拡張

**理由**: WWDC26 で Foundation Models が multimodal + LanguageModel protocol 解禁。on-device で **API キー不要・コスト ¥0** で動的お題語彙生成が成立。Apple モデル不足時は Claude API に同じコードで切替可能（`api-key-policy.md` 第 2 条遵守）。

仕様：
- 既存 256 お題をベースに、人数 / 関係性 / 場面で**語彙だけ**差し替えた変奏を生成
- iPhone 15 Pro 以降のみ「無限お題モード」追加課金（v1.2 候補）
- 非対応端末は固定 256 お題のフォールバック維持で互換性温存
- 17+ 維持（自由生成は禁止・既存お題テンプレへの語彙差し込みのみ）

### 4. Featuring Nomination 第 2 弾提出

**理由**: iOS 27 GA 当日 Editorial 露出狙い。Apple Games アプリの Featuring Nomination は **indie 平均 DL lift +792-1,747%**。

スケジュール：
- v1.0 リリース後 60 日経過時点で第 2 弾提出
- 12 月忘年会キャンペーン向け asset 入稿（Asset Library live 更新）
- Innovation 軸（App Intents + Live Activity + Foundation Models）+ Accessibility 軸（Dynamic Type 300% / VoiceOver / Family Sharing）の 2 軸プッシュ

### 5. Family Sharing 対応強化

**理由**: 3 世代帰省は年 2 回（お盆・正月）で **8 割集合**。70 代後半でアプリ DL 率 4 割未満 → 孫の親世代（30-50 代）が DL → Family Sharing で実家世代に共有が現実解。

仕様：
- 既存買切 IAP（v1.0 で実装する場合）に Family Sharing 対応設定
- App Store メタデータと LP に「Apple Family Sharing 対応、1 回購入で実家のおじいちゃんおばあちゃんも遊べます」を明記
- Dynamic Type 300% / 最小タップ 44pt / VoiceOver 対応を Accessibility 軸の Featuring 弾に転用

## 含めない（v1.2 以降に持ち越し）

| 機能 | 理由 |
|---|---|
| AI ホスト型（Aippy / Whisper Games 追従） | 17+ → 18+ アップグレードリスク・AdMob 収益崩壊 |
| ユーザー投稿型（UGC） | 18+ 確実・モデレーション工数増 |
| マルチプレイ / WebRTC | Bunch / Marco Polo / Houseparty 死因の「リモート前提」に該当 |
| Discord Activities Web 版 | 完全オフライン要件と矛盾・別プロダクトとして分離 |
| 後付けペイウォール | Bunch / Picolo 死因・信頼貯金破綻 |
