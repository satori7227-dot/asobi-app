Phase 1. コード変更後（毎回）

  • ☐ xcodebuild -project Asobi.xcodeproj -scheme Asobi -destination 'platform=iOS Simulator,name=iPhone 17' build 成功
  • ☐ fastlane test（XCUnitTest 緑）
  • ☐ swiftlint エラー0（warning は許容、要因は // swiftlint:disable で明示）
  • ☐ git status でコミット忘れ確認

Phase 2. Instruments 計測（リリース前 / 月次）

実機 iPhone（Release ビルド）で必ず実施。Simulator は数値が信頼できない。

テンプレシナリオゲート
App Launchcold start → ScenePicker 表示< 400ms（実機 A18）
Time Profiler + os_signpost起動経路。Points of Interest に load games.json 区間が出る（AsobiSignpost 実装済）load < 60ms 目安
Time Profiler13シーン × 連続シャッフル20回main thread 60FPS 維持
Allocations5分間スワイプ・シャッフル操作peak RAM < 80MB
Hangs初回起動 → 提案 → 詳細Hang 0件（>100ms）

Instruments → Time Profiler テンプレに「Points of Interest」インストゥルメントを追加すると、 AsobiSignpost.begin/end("load games.json") の区間が時間軸に表示される。 この実測値で GameRepository の同期ロード継続 or GameIndex 分離（保留中）を判断する。

Phase 3. Accessibility Audit（リリース前）

  • ☐ Accessibility Inspector の Audit を全画面で実行 → 残課題ゼロ
  • ☐ VoiceOver で「シーン選択 → 提案 → 詳細 → お気に入り」の主要動線を音声のみで完走
  • ☐ Dynamic Type を AccessibilityXXXL に設定 → レイアウト破綻なし
  • ☐ Reduce Motion ON でアニメ短縮 / Reduce Transparency ON で背景塗りつぶし
  • ☐ Smart Invert ON で色反転崩れチェック

Phase 4. App Store Connect メタデータ（変更時）

  • ☐ Privacy Policy URL がリンク切れでない（GitHub Pages）
  • ☐ Support URL が応答可能
  • ☐ App Privacy = Data Not Collected と実態が一致
  • ☐ Age Rating 12+ 維持
  • ☐ スクリーンショット（iPhone 16/17 Pro Max + 6.5"）最新
  • ☐ Promotional Text（無審査・即時反映）= 季節キャンペーン更新
  • ☐ What's New = 日本語＋英語（後者は省略可）

Phase 5. TestFlight 配信

  • ☐ fastlane bump で CURRENT_PROJECT_VERSION 更新
  • ☐ fastlane beta 成功 → App Store Connect 反映確認（5-10分）
  • ☐ 内部テスター（自分）で実機検証：起動 / 1176件読込 / シャッフル / お気に入り / 意見箱
  • ☐ Beta App Review（外部テスター追加時は初回 24h 程度）
  • ☐ Public Link 経由なら r/TestFlight / r/iOSBeta に投稿（顔出しなしテキストのみ）

Phase 6. App Store 申請

  • ☐ Phased Release ON（1日ごとに 1→2→5→10→20→50→100%）
  • ☐ Submit for Review（手動。初回は自動 release ではなく Manual Release 推奨）
  • ☐ Resolution Center 開きっぱなしで通知監視（24h 以内応答）

Phase 7. リリース後（24時間 / 1週間 / 1ヶ月）

24時間以内

  • ☐ App Store Connect Analytics でクラッシュ率確認
  • ☐ Phased Release のロールアウト率を毎朝記録
  • ☐ X / note で告知投稿（feedback_publish_includes_announcement 準拠）

1週間以内

  • ☐ レビュー★4.0 以上維持確認
  • ☐ Discord 意見箱の新規投稿全件目視
  • ☐ DL数・有料変換率を docs/metrics-log.md に追記

1ヶ月以内

  • ☐ Promotional Text 月次差し替え（季節文脈）
  • ☐ Featuring Nomination を次バージョン分で提出
  • ☐ Product Page Optimization の A/B テスト勝者反映

Phase 8. リリース後の振り返り（必須）

毎リリース後、以下を docs/release-log/YYYY-MM-DD-vX.Y.Z.md に残す（Rule 10 + Rule 13）。

  • やったこと（コード変更・メタデータ変更）
  • 判断に迷った点（採用した選択肢と棄却理由）
  • 仕様外で決めた点（Implementation Notes）
  • 次回への申し送り

緊急時：Phased Release 中の致命バグ発見

  • ☐ App Store Connect → Version → Phased Release → Pause（最大 30 日まで保留可）
  • ☐ Hotfix ブランチで修正 → fastlane beta 即座に流す
  • ☐ App Store Connect で「Expedited Review」リクエスト（年数回まで使える）
  • ☐ ユーザーには What's New 欄で透明な変更点を明示
