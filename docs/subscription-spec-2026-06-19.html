6. リスクと緩和

サブスク疲れ・解約率高い
パーティーゲームは頻繁には使わないため、月額より年額・買い切り誘導を主導線に。月額は「お試し」位置付けで価値を理解してから年額へ。
ヘルメスAI のコスト超過
月50回 制限で API コストを管理。Claude Haiku で1回 ¥0.3〜0.5 想定 → 50回 ¥15-25 × ユーザー数。サブスク粗利 ¥108/月から十分カバー。
memory project_haruhi_api_billing の知見：spend limit + ローテートでコスト上限管理。
買い切り¥1,200 と年額¥1,100 の同水準
意図的に近い価格設定。買い切り「迷いユーザー受け皿」、年額「継続価値ユーザー（新規追加コンテンツ・AI機能ほしい層）」で棲み分け。差別化のためサブスクにのみヘルメスAIと自動コンテンツ追加を集中させる。

7. StoreKit 実装の技術構成

  • StoreKit 2（iOS 15+ の最新版）を採用
  • サブスクは Product.SubscriptionInfo で月額・年額 1 グループとして登録
  • Transaction.currentEntitlements をリッスンして購入状態リアルタイム反映
  • サーバー側 receipt 検証は v1 では省略（StoreKit 2 はクライアント側 JWS 検証で十分）、後で App Store Server API 連携
  • App Store Connect で com.idogawa.Asobi.premium.monthly, .yearly, .allin を登録
  • 無料トライアル 7 日 を年額に付与（解約率を低減）
